--------------------------
跨机器工作连续性手册   
--------------------------

  核心原则                                                                      
  git 是唯一可靠的跨机器载体。不在 git 里的东西，换电脑就丢。                   
  ---
  三个文件，各司其职

  repo/
  ├── CLAUDE.md              # 项目规则 + 当前状态（自动加载）
  ├── .claude/MEMORY.md      # 记忆种子（换机器时手动注入一次）
  └── mc-portfolio.md        # 决策日志（手动 @ 引用）

  CLAUDE.md — 写什么：
  - 当前最优配置参数（数字、阈值）
  - 不能从代码推断的判断结论
  - 开发约定和踩坑记录
  - 指向其他文档的索引

  不要写：函数行号、代码结构、可以实时扫描到的内容

  ---
  日常习惯（每次重大改动后）

  1. 更新 CLAUDE.md（如果策略参数或规则变了）
  2. 更新 .claude/MEMORY.md（如果有新的经验教训）
  3. 更新 mc-portfolio.md（如果有新的金融决策）
  4. git commit + push（三个文件一起）

  可以用 /md-update 命令让 Claude 自动总结并更新。

  ---
  换新电脑的标准流程

  # 第 1 步：克隆仓库
  git clone <your-repo-url>

  # 第 2 步：安装依赖
  pip install riskfolio-lib numpy pandas xlwings akshare

  # 第 3 步：注入记忆（对 Claude 说一句话）
  "读取 .claude/MEMORY.md，写入你的系统记忆"

  # 完成，CLAUDE.md 已自动加载，工作可立即继续

  ---
  Token 经济性原则

  ┌──────────────────────┬────────────────────────────────────────┐
  │         场景         │                  做法                  │
  ├──────────────────────┼────────────────────────────────────────┤
  │ 新对话，做金融分析   │ @mc-portfolio.md                       │
  ├──────────────────────┼────────────────────────────────────────┤
  │ 新对话，改代码       │ 直接描述任务，让 Claude 实时扫描       │
  ├──────────────────────┼────────────────────────────────────────┤
  │ 新对话，改稳健性参数 │ CLAUDE.md 已有足够上下文，无需额外操作 │
  ├──────────────────────┼────────────────────────────────────────┤
  │ 完全陌生的新人接手   │ CLAUDE.md + mc-portfolio.md 全给       │
  └──────────────────────┴────────────────────────────────────────┘

  记住：CLAUDE.md 精准 → Claude 少读代码 → 省 token。文档写得越烂，Claude
  越需要自己去扫描源码弥补上下文。

  ---
  一句话总结

  CLAUDE.md 写结论，mc-portfolio.md 写过程，.claude/MEMORY.md 做备份，三者
  commit 进 git，换电脑三步走。
